# /safe-commit — 안전 자동 커밋 (push 없음)

작업한 변경사항을 검증하고 커밋한다. **push는 안 함** — 원격 저장소 없어도 동작.

## 1. 사전 안전 검증
- `.git/` 존재 확인. 없으면 **즉시 중단**, `/setup-git` 먼저 실행 안내
- `git status`로 변경 파일 목록 확인. 변경 없으면 "커밋할 변경사항 없음" 출력 후 중단
- 현재 브랜치 확인 → `main`/`master`면 경고만 출력 (학습 단계에선 진행 허용)

## 2. 시크릿 차단 (CLAUDE.md 절대 규칙)
변경 파일 중 다음에 해당하면 **즉시 중단**:
- `.env` (단 `.env.example`은 허용)
- `*.key`, `*.pem`
- `secrets/` 하위 모든 파일

추가로, 변경 파일 내용에 다음 패턴이 평문으로 보이면 경고 후 사용자 확인:
- `KIS_*_APP_KEY=` 뒤에 비어있지 않은 값
- `KIS_*_APP_SECRET=` 뒤에 비어있지 않은 값
- `ANTHROPIC_API_KEY=` 뒤에 비어있지 않은 값

## 3. 테스트 실행 (존재하면만)
- `backend/tests/` 폴더가 있으면 `cd backend && pytest` 실행. 실패 시 **즉시 중단**, 에러 로그 표시
- `frontend/` 안에 테스트 설정이 있으면 `cd frontend && pnpm test`. 실패 시 동일하게 중단
- 어느 쪽도 없으면 "테스트 없음 — 스킵" 출력하고 계속

## 4. 변경 분석 + 커밋 메시지 자동 작성
- `git diff` 결과 분석
- 우리 컨벤션 `<type>: <한국어 설명>` 형식으로 메시지 작성
- type 자동 분류 가이드:
  - 새 기능·새 파일 → `feat`
  - 버그 수정 → `fix`
  - 리팩토링(동작 동일) → `refactor`
  - 문서·주석만 변경 → `docs`
  - 테스트 추가·수정 → `test`
  - 설정·의존성·기타 → `chore`

## 5. 사용자 승인
작성된 메시지를 사용자에게 보여주고 진행 여부를 묻는다:
```
제안 커밋 메시지: "feat: 코스피200 시세 수집기 추가"
변경 파일: 3개 (backend/app/market/krx.py, ...)
진행할까요? (yes/no/edit)
```
- `no` → 중단
- `edit` → 사용자가 직접 메시지 입력
- `yes` → 다음 단계

## 6. 커밋
- `git add -A`
- staging 후 한 번 더 시크릿 더블체크 (2번 단계 재실행)
- `git commit -m "<승인된 메시지>"`

## 7. 결과 요약
- 커밋 해시 (짧은 형식)
- 커밋 메시지
- 변경 파일 수
- 다음 단계: "push 하려면 `/ship` 또는 수동 `git push origin <브랜치>`"
